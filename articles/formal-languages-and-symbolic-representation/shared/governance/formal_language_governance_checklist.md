# Formal Language Governance Checklist

- Define the representation purpose and scope.
- Define allowed symbols, tokens, encodings, and reserved forms.
- Specify grammar, schema, or syntax rules explicitly.
- Separate syntax validity from semantic adequacy.
- Provide positive and negative examples.
- Build useful error messages for invalid structures.
- Version grammar, schemas, tokens, and compatibility rules.
- Test ordinary cases, edge cases, malformed inputs, and ambiguity.
- Document interpretation limits and responsible-use conditions.
- Review downstream consequences when symbolic categories change.
