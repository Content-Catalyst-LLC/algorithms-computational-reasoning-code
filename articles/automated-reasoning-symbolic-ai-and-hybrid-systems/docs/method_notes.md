# Method Notes

This folder treats automated reasoning as a reviewable workflow.

The core method is:

1. Define the reasoning task.
2. Specify facts, rules, constraints, and assumptions.
3. Run inference or solving.
4. Preserve inference traces.
5. Check contradictions and missing constraints.
6. Review rule provenance.
7. Define human review, appeal, and use boundaries.

The examples are intentionally dependency-light. They demonstrate the structure of symbolic audit records rather than production theorem proving.
