# Validation Evidence Cards

## Intended use
What claim, decision, or interpretation will the model support?

## Data validation
What input-quality evidence supports the model run?

## Implementation testing
What tests show that code implements the intended method?

## Benchmark comparison
What known cases, baselines, or references anchor credibility?

## Diagnostics
Where does the model work, fail, drift, or show bias?

## Governance
Who reviews, approves, monitors, and revalidates use?
