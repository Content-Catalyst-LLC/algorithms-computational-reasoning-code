# Halting Boundary Governance Checklist

- State the exact automation claim.
- Define the program language, model, workflow, or system scope.
- Define the property being checked.
- Distinguish termination, safety, liveness, correctness, and equivalence claims.
- State restrictions, assumptions, annotations, and boundedness.
- Distinguish yes, no, unknown, timeout, out-of-scope, and requires-review states.
- Preserve counterexamples, traces, diagnostics, and proof artifacts.
- Avoid treating timeouts as proof of nontermination.
- Avoid treating AI-generated explanations as universal decision procedures.
- Provide escalation paths for high-stakes, unresolved, or ambiguous cases.
