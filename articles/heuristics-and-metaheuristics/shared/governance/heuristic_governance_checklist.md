# Heuristic Governance Checklist

- Define the problem, heuristic rule, score, move set, or search framework.
- State what the heuristic is meant to optimize or improve.
- Document parameters, seeds, stopping rules, and default settings.
- Compare performance against baselines, exact small cases, and known alternatives.
- Stress test edge cases, adversarial cases, subgroup impacts, and rare failures.
- Run multi-seed and parameter sensitivity checks for randomized heuristics.
- Preserve logs, scores, parameters, versions, seeds, and benchmark records.
- Monitor drift after deployment and define review cadence.
- Label heuristic outputs as heuristic, approximate, or non-guaranteed where appropriate.
- Assign responsibility for accepting heuristic risk.
