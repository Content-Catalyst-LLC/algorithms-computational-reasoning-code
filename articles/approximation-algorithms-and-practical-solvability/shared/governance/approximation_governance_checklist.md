# Approximation Governance Checklist

- Define the exact problem, objective, and constraints.
- Identify why exact optimization is impractical.
- State the approximation method and whether it is greedy, relaxation-based, randomized, scheme-based, or heuristic.
- Prove or document feasibility preservation.
- State the approximation ratio, additive error, optimality gap, empirical gap, or probability-bound claim.
- State assumptions under which the guarantee applies.
- Translate approximation gaps into domain consequences.
- Test edge cases, rare cases, and subgroup impacts.
- Preserve run logs, bounds, relaxed values, rounded values, seeds, and transformation records.
- Assign decision authority for accepting the accuracy-cost trade-off.
