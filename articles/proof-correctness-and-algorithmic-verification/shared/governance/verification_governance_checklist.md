# Verification Governance Checklist

- State the correctness claim explicitly.
- Define preconditions and input scope.
- Define postconditions and output relation.
- Identify invariants and safety properties.
- Address termination and progress.
- Preserve test cases, proof notes, and counterexamples.
- Connect specification, code, generated outputs, and release state.
- Distinguish verified claims from real-world adequacy.
- Review assumptions when context, data, rules, or goals change.
- Keep human accountability attached to consequential systems.
