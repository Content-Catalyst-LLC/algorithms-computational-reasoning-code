# Method Notes

This folder treats Dijkstra as a theorist of structured programming, correctness, invariants, weakest preconditions, guarded commands, and disciplined software reasoning.

The core method is:

1. Begin with the problem of unstructured control flow.
2. Read the go to debate as a reasoning problem, not only a syntax issue.
3. Study sequence, selection, iteration, blocks, and procedures.
4. Identify preconditions, postconditions, invariants, and termination arguments.
5. Use weakest-precondition reasoning to work backward from results.
6. Study guarded commands as formal control structures.
7. Analyze Dijkstra's algorithm through assumptions and invariants.
8. Apply structured reasoning to AI-generated code and modern systems.

The goal is to understand programming as disciplined reasoning, not mere execution.
