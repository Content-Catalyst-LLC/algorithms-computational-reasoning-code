# AI Code Review Notes

Dijkstra's discipline applies directly to AI-generated code.

Review generated code by asking:

- What is the specification?
- What preconditions are assumed?
- What postconditions should hold?
- What invariants are required?
- What edge cases are excluded?
- Does the structure support human reasoning?
- What tests cover the stated obligations?
- Who remains accountable for the code?

Plausible code is not the same as correct code.
