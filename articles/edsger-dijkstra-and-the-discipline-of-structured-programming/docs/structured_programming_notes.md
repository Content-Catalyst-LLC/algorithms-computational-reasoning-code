# Structured Programming Notes

Structured programming organizes control flow through:

- sequence;
- selection;
- iteration;
- blocks;
- procedures;
- modules.

The purpose is not merely style. Structured control supports reasoning about program behavior.

A structured program should help a reader answer:

- What can happen next?
- What state changes here?
- What assumptions are required?
- What property is preserved?
- Why does the program terminate?
- Why does the final result satisfy the specification?
