# Edsger Dijkstra and the Discipline of Structured Programming

This companion folder supports the Sustainable Catalyst article **Edsger Dijkstra and the Discipline of Structured Programming**.

It provides reproducible, teaching-oriented workflows for studying Edsger Dijkstra's programming discipline: structured programming, go to statements, control flow, sequence, selection, iteration, program correctness, preconditions, postconditions, invariants, weakest preconditions, guarded commands, predicate transformers, stepwise refinement, program derivation, Dijkstra's shortest-path algorithm, layered operating systems, semaphores, concurrency, formal methods, software engineering, AI code-generation review, and responsible structured reasoning.

## What this folder demonstrates

- how structured programming replaced arbitrary jumps with disciplined control flow;
- how sequence, selection, iteration, procedures, and modules support human reasoning;
- how preconditions, postconditions, invariants, and termination arguments support correctness;
- how weakest preconditions and guarded commands treat programming as formal derivation;
- how Dijkstra's shortest-path algorithm illustrates assumptions, invariants, and proof;
- how layered operating systems and concurrency require structured system reasoning;
- why testing is necessary but not the same as proof;
- how Dijkstra's discipline applies to AI-generated code, review, and responsible automation.

## Repository structure

- `python/` — reference structured-programming discipline map workflow and tests
- `r/` — structured-programming diagnostics and figures
- `sql/` — schema and queries for Dijkstra themes, scores, and cautions
- `calculators/` — self-contained discipline-score, invariant-check, weakest-precondition, Dijkstra shortest-path, control-flow complexity, loop-termination, proof-obligation, and AI-generated-code caution calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, structured programming notes, invariant notes, weakest precondition notes, Dijkstra algorithm notes, AI code-review notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not replacements for formal verification, compiler theory, program logic, security review, or production software certification.
