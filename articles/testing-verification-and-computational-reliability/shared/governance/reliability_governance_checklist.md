# Reliability Governance Checklist

- Define expected behavior and specification boundaries.
- Identify important risks, failure modes, edge cases, and abuse cases.
- Choose unit, integration, system, acceptance, security, and reliability tests based on risk.
- Define test oracles, tolerances, invariants, and expected errors.
- Convert regressions and incidents into repeatable tests.
- Preserve reproducibility evidence: source version, data snapshot, environment, seed, outputs, and logs.
- Monitor runtime behavior with logs, metrics, traces, alerts, and audit records.
- Assign ownership for tests, incidents, risk acceptance, and retirement.
- Review reliability evidence before consequential deployment.
