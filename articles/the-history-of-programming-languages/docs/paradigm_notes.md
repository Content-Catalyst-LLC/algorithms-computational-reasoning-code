# Paradigm Notes

Programming paradigms organize how computation is expressed.

Common paradigms:

- imperative: commands update state;
- procedural: procedures organize steps;
- functional: functions and expressions compose;
- object-oriented: objects encapsulate state and behavior;
- logic: rules and facts support inference;
- declarative: state what is wanted rather than every control step;
- dataflow: computation follows data dependencies;
- concurrent/actor: many processes interact.

Paradigms are not merely categories. They shape reasoning, error patterns, software architecture, and pedagogy.
