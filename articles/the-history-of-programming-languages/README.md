# The History of Programming Languages

This companion folder supports the Sustainable Catalyst article **The History of Programming Languages**.

It provides reproducible, teaching-oriented workflows for studying programming-language history as a layered history of expression, execution, abstraction, ecosystems, institutions, safety, and governance. It covers assembly, automatic coding, Fortran, Lisp, ALGOL, COBOL, BASIC, Pascal, C, Simula, Smalltalk, Prolog, ML, SQL, Ada, C++, Java, JavaScript, Python, R, Haskell, Rust, Julia, TypeScript, domain-specific languages, compilers, interpreters, runtimes, type systems, package ecosystems, and AI code-generation review.

## What this folder demonstrates

- how programming languages mediate between human-readable source and machine behavior;
- why language history is not simple progress from old to new;
- how paradigms shape control, data, abstraction, error, and reasoning;
- how language ecosystems include compilers, interpreters, runtimes, libraries, packages, tools, standards, and communities;
- why Fortran, Lisp, ALGOL, COBOL, C, SQL, Python, R, Haskell, Rust, JavaScript, and other languages solved different problems under different constraints;
- how AI code generation adds another translation layer without ending language history;
- why generated code still depends on language semantics, tooling, dependencies, tests, documentation, and governance.

## Repository structure

- `python/` — reference programming-language history map workflow and tests
- `r/` — language-history diagnostics and figures
- `sql/` — schema and queries for language traditions, scores, and cautions
- `calculators/` — self-contained language-history score, paradigm, ecosystem, type-safety, translation-risk, dependency-risk, and AI-code governance calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, language ecosystem notes, paradigm notes, type-system notes, AI review notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not language benchmarks, security audits, compiler conformance tests, or software certification.
