# Method Notes

This folder treats documentation as governance infrastructure.

The core method is:

1. Define purpose, intended use, decision role, and prohibited use.
2. Document data origin, composition, collection, labels, transformations, correction, and maintenance.
3. Document model details, evaluation, metrics, subgroup performance, limitations, and failure modes.
4. Link model cards and datasheets to risk registers, control registers, owners, and monitoring plans.
5. Record versions, changes, thresholds, prompts, deployment conditions, and governance approvals.
6. Document notice, reasons, evidence access, appeal, correction, and remedy pathways.
7. Score completeness, artifact coverage, documentation quality, freshness, and documentation risk.

The examples are dependency-light and educational.
