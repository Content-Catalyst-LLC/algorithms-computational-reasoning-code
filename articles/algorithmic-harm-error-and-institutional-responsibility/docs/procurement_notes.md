# Procurement Notes

Procurement is governance.

When an institution purchases, licenses, or integrates an algorithmic system, it should require:

- model documentation;
- data provenance and quality evidence;
- audit rights;
- evaluation evidence for the intended context;
- update and version-control requirements;
- incident reporting obligations;
- appeal and reason-code support;
- rollback and termination terms;
- remediation responsibilities;
- clear allocation of vendor and deploying-institution duties.

A deploying institution remains responsible for the system it uses.
