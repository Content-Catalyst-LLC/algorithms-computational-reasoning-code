# Method Notes

This folder treats algorithmic harm as a socio-technical and institutional issue.

The core method is:

1. Identify the decision, workflow, and affected parties.
2. Classify harm type and severity.
3. Trace error sources across data, model, interface, workflow, policy, and governance.
4. Evaluate exposure and contestability.
5. Map institutional responsibility across ownership, monitoring, appeal, repair, and governance.
6. Record incidents and near misses.
7. Remediate individual outcomes and source conditions.
8. Monitor recurrence and update governance.

The examples are dependency-light and educational.
