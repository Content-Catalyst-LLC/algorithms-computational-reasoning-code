# Incident Notes

An algorithmic incident is an event where a system produces or contributes to harm, near harm, unexpected behavior, policy violation, unfair treatment, safety risk, or loss of trust.

A useful incident record should include:

- system identifier and version;
- decision context;
- harm type;
- severity;
- affected person, group, context, or workflow;
- data inputs and output records;
- human actions and reliance behavior;
- appeal, override, and review history;
- remediation status;
- recurrence monitoring.

Incident reporting converts anecdotal harm into governance evidence.
