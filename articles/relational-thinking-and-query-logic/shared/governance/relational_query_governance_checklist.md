# Relational Query Logic Governance Checklist

- Translate every query into a plain-language question.
- Identify entities, attributes, relationships, keys, and predicates.
- Validate join logic and relationship cardinality.
- Check duplicate multiplication and accidental omission.
- Distinguish null, unknown, not applicable, withheld, and absent relationships.
- Review anti-joins for missing required evidence.
- Define groups, denominators, thresholds, and aggregations.
- Preserve query text, parameters, execution context, and versions.
- Review access controls and safe fields.
- Connect query results to provenance, lineage, audit trails, and interpretation notes.
