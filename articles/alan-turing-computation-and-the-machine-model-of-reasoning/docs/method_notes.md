# Method Notes

This folder treats Turing as a bridge between effective procedure, machine abstraction, formal limits, and machine-intelligence debate.

The core method is:

1. Separate Turing's 1936 computability work from his 1950 machine-intelligence work.
2. Define effective procedure carefully.
3. Trace the machine model: tape, symbols, head, states, and transitions.
4. Analyze universal computation as simulation from description.
5. Study undecidability as a limit, not a failure.
6. Distinguish computability from efficiency and feasibility.
7. Read the imitation game as behavioral framing, not a full theory of mind.
8. Connect Turing's work to responsible automation and human judgment.

The goal is to understand Turing's machine model without turning it into either limitless automation or a total model of all reasoning.
