# Alan Turing, Computation, and the Machine Model of Reasoning

This companion folder supports the Sustainable Catalyst article **Alan Turing, Computation, and the Machine Model of Reasoning**.

It provides reproducible, teaching-oriented workflows for studying Turing's machine model of reasoning: effective procedure, computable numbers, finite means, Turing machines, tape, symbols, states, transition rules, human computers, universal machines, program-as-description, decidability, undecidability, the Entscheidungsproblem, the halting problem, Church-Turing thesis, machine intelligence, imitation game, AI caution, and responsible algorithmic reasoning.

## What this folder demonstrates

- how Turing formalized effective procedure;
- how tape, symbols, states, head position, and transition rules model computation;
- how a human calculator following definite rules becomes an abstract machine model;
- how universal machines make program-as-description thinkable;
- how undecidability and the halting problem establish formal limits of automation;
- how Turing relates to Church, lambda calculus, and the Church-Turing thesis;
- how the imitation game reframes machine intelligence without settling every AI question;
- how Turing's legacy supports both computational ambition and disciplined caution.

## Repository structure

- `python/` — reference Turing machine reasoning map workflow and tests
- `r/` — computability diagnostics and figures
- `sql/` — schema and queries for Turing themes, scores, and cautions
- `calculators/` — self-contained Turing-machine, transition, halting, universality, decidability, imitation-game, and AI-caution calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, Turing machine notes, universality notes, undecidability notes, AI caution notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and interpretive. They are educational scaffolds for historical and conceptual analysis, not replacements for formal computability theory, proof, or specialist history-of-computing scholarship.
