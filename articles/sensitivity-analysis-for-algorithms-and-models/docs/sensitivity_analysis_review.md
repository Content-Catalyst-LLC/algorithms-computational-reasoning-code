# Sensitivity Analysis Review Notes

A defensible sensitivity analysis should document:

1. Baseline configuration.
2. Inputs, parameters, thresholds, assumptions, and model structures that may vary.
3. Range justification.
4. One-at-a-time and scenario sweeps.
5. Threshold sensitivity.
6. Seed sensitivity for stochastic workflows.
7. Structural sensitivity where model form is contested.
8. Governance implications for monitoring, validation, and decision use.
