# Agent-Based Algorithms Review Checklist

Use this checklist when reviewing an agent-based algorithm or model.

1. Define the modeling purpose and intended use.
2. Define agents, attributes, and system boundaries.
3. Document behavior rules and their evidence or rationale.
4. Document environment structure: grid, network, market, landscape, or institution.
5. Specify interaction rules, neighborhoods, contacts, and update order.
6. Record stochastic assumptions, seeds, sample sizes, and repeated-run design.
7. Run sensitivity tests across rules, thresholds, parameters, networks, and initial conditions.
8. Compare outputs with observed patterns, theory, benchmarks, or expert expectations.
9. Preserve code, data, parameters, seeds, outputs, and environment records.
10. State representation limits and governance boundaries.
