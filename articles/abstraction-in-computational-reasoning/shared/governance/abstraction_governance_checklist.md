# Abstraction Governance Checklist

- Name the real system, process, or concern being abstracted.
- Define the purpose of the abstraction.
- Identify the representation type: variable, function, table, graph, schema, model, score, interface, or type.
- Document preserved features and suppressed details.
- Define scope and invalid use cases.
- Record assumptions, uncertainty, provenance, and limitation notes.
- Test whether the abstraction supports the intended reasoning task.
- Explain how outputs should and should not be interpreted.
- Review reuse safety before applying the abstraction in a new context.
- Monitor for context loss, false precision, proxy confusion, and responsibility hiding.
