# John von Neumann and the Architecture of Modern Computing

This companion folder supports the Sustainable Catalyst article **John von Neumann and the Architecture of Modern Computing**.

It provides reproducible, teaching-oriented workflows for studying von Neumann's architectural role in modern computing: stored-program architecture, EDVAC, ENIAC, IAS machine, memory, instructions, data, control units, arithmetic units, input/output, sequential execution, branching, loops, program-as-data, self-modifying programs, the von Neumann bottleneck, architecture vs implementation, software layers, scientific computing, AI infrastructure, credit disputes, and responsible architecture governance.

## What this folder demonstrates

- how stored-program computing made software a durable layer of computation;
- how instructions and data became symbolic objects stored in memory;
- how control, arithmetic, memory, input, and output form a general architecture;
- how program-as-data enables compilers, interpreters, operating systems, code generation, and security risks;
- how the von Neumann bottleneck links architecture to performance, energy, and AI infrastructure;
- how architecture differs from implementation;
- how credit around stored-program computing requires collaborative and institutional framing;
- how responsible computing requires architectural understanding, not only algorithmic analysis.

## Repository structure

- `python/` — reference von Neumann architecture map workflow and tests
- `r/` — architecture diagnostics and figures
- `sql/` — schema and queries for architecture themes, scores, and cautions
- `calculators/` — self-contained architecture-score, fetch-execute, memory-bandwidth, bottleneck, program-as-data, branch, locality, and governance calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, stored-program notes, EDVAC notes, bottleneck notes, software-layer notes, governance notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and interpretive. They are educational scaffolds for historical and conceptual analysis, not replacements for computer architecture, hardware engineering, or specialist history-of-computing scholarship.
