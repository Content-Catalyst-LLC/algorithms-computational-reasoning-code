# Method Notes

This folder treats von Neumann as an architectural synthesizer of stored-program computing, not a lone inventor.

The core method is:

1. Separate the stored-program idea from lone-inventor mythology.
2. Read the EDVAC draft as architectural synthesis and vocabulary.
3. Identify memory, control, arithmetic, input, and output.
4. Trace how instructions and data share memory.
5. Model fetch-decode-execute and control flow.
6. Analyze the von Neumann bottleneck.
7. Compare von Neumann, Harvard, modified Harvard, GPU, vector, and distributed architectures.
8. Connect architecture to software, AI infrastructure, security, and governance.

The goal is to understand architecture as a technical, historical, institutional, and governance structure.
