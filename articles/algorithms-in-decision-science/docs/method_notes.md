# Method Notes

This folder treats decision algorithms as accountable support systems rather than automatic authorities.

The core method is:

1. Define the decision, action options, stakes, affected people, and authority.
2. Identify the forecast, score, model output, or rule needed for decision support.
3. Document thresholds, expected value, expected loss, constraints, and escalation rules.
4. Evaluate calibration, uncertainty, subgroup performance, robustness, and model limits.
5. Define human review, override, contestability, and remediation.
6. Monitor outcomes, reliance, feedback effects, drift, and unintended consequences.
7. Revise, limit, pause, or retire the system when evidence changes.

The examples are dependency-light and educational.
