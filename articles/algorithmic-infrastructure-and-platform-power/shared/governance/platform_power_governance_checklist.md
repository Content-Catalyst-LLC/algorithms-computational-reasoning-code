# Algorithmic Infrastructure and Platform Power Governance Checklist

- Map platform dependencies across search, cloud, APIs, marketplaces, identity, payments, AI platforms, analytics, and data pipelines.
- Identify access chokepoints: account access, API keys, app review, seller access, payment approval, model endpoints.
- Evaluate visibility dependence: search ranking, feed placement, marketplace ordering, recommender systems, ad auctions.
- Review data extraction and measurement: telemetry, behavioral data, seller metrics, developer metrics, risk scores, model feedback.
- Quantify switching costs: migration, rebuild, training, downtime, lost network effects, data export, and contract dependence.
- Review interoperability and portability: open formats, export quality, abstraction layers, fallback systems, standards.
- Preserve audit evidence: access decisions, ranking changes, API changes, billing changes, model version changes, moderation actions.
- Define appeal and remediation paths for consequential access, visibility, payment, moderation, ranking, or model decisions.
- Establish governance ownership for platform-risk review, dependency monitoring, and public-interest consequences.
- Communicate uncertainty, dependency, and limits clearly to affected participants and institutional stakeholders.
