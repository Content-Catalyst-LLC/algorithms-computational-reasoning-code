# Database Knowledge System Governance Checklist

- Define the knowledge purpose and institutional use.
- Inventory entities, relationships, fields, keys, and constraints.
- Review schema clarity, category definitions, missingness, and units.
- Document queries the system can answer and questions it cannot answer.
- Review indexes as evidence of retrieval priorities.
- Validate transaction reliability, concurrency expectations, and audit logs.
- Preserve metadata, provenance, lineage, versioning, and transformation records.
- Define access, permissions, exports, deletion, correction, and appeal workflows.
- Review retention, archiving, deletion, and evidence-preservation requirements.
- Assess representation risk: what the database clarifies, simplifies, and omits.
