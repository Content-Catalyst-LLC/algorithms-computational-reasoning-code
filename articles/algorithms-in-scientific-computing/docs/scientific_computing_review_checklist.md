# Scientific Computing Review Checklist

Use this checklist when reviewing algorithms in scientific computing.

1. Define the scientific question, intended use, and decision context.
2. Document the mathematical model, variables, assumptions, units, and boundaries.
3. Justify the numerical method, solver, step size, grid, tolerance, and stopping rule.
4. Check finite precision, roundoff, cancellation, overflow, underflow, and stability risk.
5. Run convergence or resolution tests where numerical approximation matters.
6. Compare against analytic solutions, benchmarks, observations, or expert expectations when possible.
7. Quantify uncertainty through parameter sweeps, Monte Carlo runs, ensembles, or sensitivity analysis.
8. Preserve code, data, parameters, environment details, seeds, logs, outputs, and provenance.
9. Communicate meaningful precision, error estimates, assumptions, and limitations.
10. Govern any decision use of computational results.
