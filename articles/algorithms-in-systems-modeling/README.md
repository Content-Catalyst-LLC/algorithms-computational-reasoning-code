# Algorithms in Systems Modeling

This companion folder supports the Sustainable Catalyst article **Algorithms in Systems Modeling**.

It provides reproducible, teaching-oriented workflows for computational systems modeling, feedback simulation, stock-and-flow updates, network dynamics, scenario modeling, agent-based reasoning, discrete-event thinking, resilience, cascading effects, sensitivity analysis, calibration, uncertainty, digital twins, governance, and responsible model interpretation.

## Article sequence

- Previous: `algorithms-in-decision-science`
- Current: `algorithms-in-systems-modeling`
- Next: `algorithms-in-knowledge-architecture`

## What this folder demonstrates

- how systems can be represented through states, flows, feedback loops, networks, scenarios, and interventions;
- how vulnerability can combine feedback strength, network dependency, uncertainty, and weak resilience;
- how readiness can combine calibration, documentation, governance, and resilience;
- how systems-modeling risk rises when high-stakes systems are vulnerable and not ready for decision use;
- how systems models should be documented with boundary statements, assumption registers, scenario archives, sensitivity analysis, uncertainty reports, decision logs, and stop rules;
- how calculators, tests, SQL queries, Canvas cards, and multilingual examples can support accountable systems modeling.

## Repository structure

- `python/` — reference systems-modeling audit workflow and tests
- `r/` — diagnostic summaries and figures
- `sql/` — schema and queries for scenarios, vulnerability, readiness, risk, and governance
- `calculators/` — self-contained stock-flow, feedback, vulnerability, readiness, sensitivity, and resilience calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, scenario notes, assumption register notes, governance review, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not legal, compliance, audit certification, civil-rights, medical, climate, public-health, infrastructure, financial, or operational advice.
