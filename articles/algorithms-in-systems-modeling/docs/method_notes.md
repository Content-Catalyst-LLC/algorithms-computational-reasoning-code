# Method Notes

This folder treats systems models as structured inquiries rather than automatic answers.

The core method is:

1. Define model purpose, decision use, system boundaries, and affected people.
2. Identify states, stocks, flows, feedback loops, networks, delays, constraints, and interventions.
3. Specify parameters, assumptions, time scales, scenarios, and external shocks.
4. Run baseline, intervention, stress, and equity scenarios.
5. Conduct sensitivity analysis, calibration, uncertainty analysis, and resilience review.
6. Document how outputs influence decisions, governance, and accountability.
7. Monitor model performance, revise assumptions, and pause or retire misleading models.

The examples are dependency-light and educational.
