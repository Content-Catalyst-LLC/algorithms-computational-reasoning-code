# Human-in-the-Loop and Human Judgment

This companion folder supports the Sustainable Catalyst article **Human-in-the-Loop and Human Judgment**.

It provides reproducible, teaching-oriented workflows for human-in-the-loop systems, human judgment, meaningful review, automation bias, overreliance, override authority, escalation, uncertainty communication, reviewer workload, interface design, contestability, governance, audit trails, remediation, and responsible automation.

## Article sequence

- Previous: `algorithmic-accountability-and-audit-trails`
- Current: `human-in-the-loop-and-human-judgment`
- Next: `responsible-automation-and-decision-delegation`

## What this folder demonstrates

- how human-in-the-loop claims can be tested against actual review conditions;
- how human review capacity can combine time, information, authority, training, and protection;
- how automation reliance and override rates can reveal rubber-stamp review;
- how review risk rises when stakes are high and judgment capacity is weak;
- how governance registers, calculators, tests, SQL queries, Canvas cards, and multilingual examples can support meaningful human judgment.

## Repository structure

- `python/` — reference human review capacity audit workflow and tests
- `r/` — diagnostic summaries and figures
- `sql/` — schema and queries for human review capacity, reliance, overrides, escalation, and governance
- `calculators/` — self-contained review-capacity, reliance, override-rate, and review-risk calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, governance review, reviewer workflow notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not legal, compliance, audit certification, civil-rights, medical, credit, employment, public-benefits, or operational advice.
