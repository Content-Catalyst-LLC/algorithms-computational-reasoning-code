# Contestability, Appeals, and Algorithmic Due Process

This companion folder supports the Sustainable Catalyst article **Contestability, Appeals, and Algorithmic Due Process**.

It provides reproducible, teaching-oriented workflows for contestability, appeals, algorithmic due process, notice, reasons, evidence access, human review, correction, remedy, appeal burden, procedural fairness, audit trails, and accountable algorithmic governance.

## Article sequence

- Previous: `automation-bias-and-human-overreliance`
- Current: `contestability-appeals-and-algorithmic-due-process`
- Next: `metrics-feedback-and-algorithmic-failure`

## What this folder demonstrates

- how algorithmic decisions can be made reviewable;
- how notice, reasons, evidence access, review authority, correction, and remedy can be represented as procedural safeguards;
- how high-stakes decisions require stronger contestability and appeal pathways;
- how appeal burden, timeliness, and reversal patterns can be monitored;
- how correction should propagate to records, decisions, policies, and model governance;
- how due process can be audited as an operational system rather than described only in policy.

## Repository structure

- `python/` — reference contestability and due-process audit workflow and tests
- `r/` — diagnostic summaries and figures
- `sql/` — schema and governance queries for appeals, review, correction, burden, and procedural risk
- `calculators/` — self-contained contestability, procedural-risk, appeal-burden, and review-trigger calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, governance review, due-process notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not legal advice and are not intended for operational certification without domain, legal, and institutional review.
