# Search Spaces and Computational Exploration Governance Checklist

- Define states, transitions, actions, initial states, and goal conditions.
- Document constraints, excluded options, infeasible branches, and pruning rules.
- Explain heuristics, scoring functions, tie-breaking rules, and search priorities.
- Preserve frontier behavior, explored-set handling, path-cost records, and repeated-state logic.
- Report search coverage, explored regions, omitted regions, and resource limits.
- Define stopping conditions: goal found, budget exhausted, confidence sufficient, or no solution.
- Preserve traces that allow the result to be reconstructed or challenged.
- Review representation risk: missing states, invalid transitions, hidden constraints, biased heuristics, premature stopping.
