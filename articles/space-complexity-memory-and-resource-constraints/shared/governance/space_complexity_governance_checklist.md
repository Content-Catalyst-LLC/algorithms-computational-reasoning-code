# Space Complexity Governance Checklist

- Define input size and all relevant input dimensions.
- Separate input space, output space, auxiliary space, total space, and peak memory.
- Identify stack, heap, cache, index, table, buffer, and intermediate-storage costs.
- Compare dense, sparse, streaming, compressed, and external-memory representations.
- Benchmark realistic and stress-test peak memory.
- Review I/O, cache behavior, serialization, copying, and network data movement.
- Define what happens when memory is exhausted.
- Include logs, audit trails, checkpoints, retention, and backups in resource planning.
- Review privacy, security, and access implications of stored data.
- Communicate memory limits and resource assumptions in plain language.
