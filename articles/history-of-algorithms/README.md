# The History of Algorithms: From Procedure to Computation

This companion folder supports the Sustainable Catalyst article **The History of Algorithms: From Procedure to Computation**.

It provides reproducible, teaching-oriented workflows for studying algorithm history as a layered movement from procedure to computation: ancient worked examples, demonstrated procedure, numeration, algebra, algorism, translation, reception, tables, instruments, symbolic notation, mechanical calculators, programmable machines, logic, computability, electronic computing, software, algorithm analysis, AI systems, and algorithmic governance.

## What this folder demonstrates

- how procedures existed before the word algorithm and before computers;
- how ancient worked examples encode reusable methods;
- how Greek geometry links procedure and proof;
- how Indian numeration, zero, and place value support written calculation;
- how Islamic-world algebra and algorism made procedures portable and teachable;
- how Latin reception, merchants, schools, and institutions spread algorism;
- how tables, instruments, and automata store or embody procedure;
- how symbolic notation, machines, logic, and computability transformed the concept of algorithm;
- how electronic computing, programming languages, data structures, complexity analysis, AI, and governance reshaped algorithmic reasoning;
- how to avoid reducing algorithm history to one person, one culture, one word, or one technology.

## Repository structure

- `python/` — reference algorithm history timeline map workflow and tests
- `r/` — algorithm history diagnostics and figures
- `sql/` — schema and queries for historical layers, scores, and cautions
- `calculators/` — self-contained procedure-score, timeline-layer, complexity, computability, governance, and history-score calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, timeline notes, procedure-to-computation notes, computability notes, governance notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and interpretive. They are educational scaffolds for historical and conceptual analysis, not claims about a single origin, lineage, or authoritative chronology.
