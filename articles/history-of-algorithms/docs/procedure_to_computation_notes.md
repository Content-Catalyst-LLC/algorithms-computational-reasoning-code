# Procedure to Computation Notes

Algorithm history can be described as the movement:

Example -> Rule -> Method -> Notation -> Machine -> Program -> System

Each transition changes what procedure can do:

- examples make procedures teachable;
- rules make procedures repeatable;
- notation makes procedures compact;
- proof makes procedures accountable;
- tables and instruments make procedures reusable;
- machines make procedures executable;
- programs make procedures scalable;
- systems make procedures institutional;
- governance makes procedures publicly responsible.
