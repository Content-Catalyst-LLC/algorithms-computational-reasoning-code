# Trade, Inheritance, Surveying, and Practical Calculation

This companion folder supports the Sustainable Catalyst article **Trade, Inheritance, Surveying, and Practical Calculation**.

It provides reproducible, teaching-oriented workflows for mapping practical calculation as institutional algorithmic reasoning: trade, inheritance, surveying, proportional allocation, commercial reckoning, land measurement, word-problem translation, written reckoning, auditability, human calculators, verification, dispute, and trust.

## What this folder demonstrates

- how practical calculation turns social, legal, commercial, and spatial situations into computable problems;
- how trade calculation, inheritance allocation, and surveying depend on repeatable procedures;
- how proportional reasoning and common measure normalize quantities for comparison and distribution;
- how written reckoning creates audit trails and institutional trust;
- how human calculators performed algorithmic work before machine computation;
- how verification, dispute, and records connect mathematics to governance;
- how practical mathematics can be sophisticated without being purely abstract.

## Repository structure

- `python/` — reference practical calculation procedure map workflow and tests
- `r/` — practical calculation diagnostics and figures
- `sql/` — schema and queries for practical-calculation themes, scores, and interpretation cautions
- `calculators/` — self-contained trade, inheritance, surveying, proportion, unit-conversion, audit-check, and practical-score calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, trade notes, inheritance notes, surveying notes, auditability notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and interpretive. They are educational scaffolds, not legal, financial, or surveying advice.
