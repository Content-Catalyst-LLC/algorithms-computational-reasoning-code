# Automation Bias and Human Overreliance

This companion folder supports the Sustainable Catalyst article **Automation Bias and Human Overreliance**.

It provides reproducible, teaching-oriented workflows for automation bias, human overreliance, automation complacency, trust calibration, algorithm aversion, human-in-the-loop limitations, interface framing, alert fatigue, override friction, appeal pathways, contestability, and oversight governance.

## Article sequence

- Previous: `distribution-shift-and-model-decay`
- Current: `automation-bias-and-human-overreliance`
- Next: `metrics-feedback-and-algorithmic-failure`

## What this folder demonstrates

- how human acceptance can exceed justified model reliability;
- how review time, uncertainty, override friction, and appeal access shape oversight;
- how human-in-the-loop systems can become rubber-stamp workflows;
- how trust calibration can be measured through reliance behavior and context-specific model quality;
- how governance records can document human authority, review conditions, training, overrides, appeals, and accountability.

## Repository structure

- `python/` — reference automation-bias and overreliance audit workflow and tests
- `r/` — diagnostic summaries and figures
- `sql/` — schema and governance queries for reliance behavior, override friction, appeal pathways, and oversight review
- `calculators/` — self-contained trust-calibration, overreliance, and oversight-risk calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, governance review, oversight notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not intended for operational oversight certification without domain validation.
