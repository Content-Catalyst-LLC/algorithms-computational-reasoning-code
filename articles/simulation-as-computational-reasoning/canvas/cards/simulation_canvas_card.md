# Simulation as Computational Reasoning Canvas Card

- Purpose: explore modeled behavior over time.
- Boundary: specify included and excluded mechanisms.
- State variables: define what changes.
- Rules: define updates, events, feedback, or agent behavior.
- Parameters: document meanings, sources, and ranges.
- Scenarios: compare baseline, intervention, stress, and counterfactual cases.
- Uncertainty: use repeated runs, sensitivity tests, and ranges.
- Evidence: connect verification, calibration, validation, and review.
- Limits: communicate representation risk and governance conditions.
