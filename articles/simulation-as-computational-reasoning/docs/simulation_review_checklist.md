# Simulation Review Checklist

Use this checklist when reviewing simulation as computational reasoning.

1. Define the simulation purpose and decision context.
2. Document model boundaries, inclusions, exclusions, and scale.
3. List entities, state variables, rules, parameters, time steps, and outputs.
4. Distinguish deterministic rules from stochastic sampling.
5. Record parameter sources, ranges, and uncertainty.
6. Run baseline, intervention, stress-test, and counterfactual scenarios.
7. Perform parameter sweeps and sensitivity analysis.
8. Run repeated stochastic simulations where randomness matters.
9. Summarize uncertainty as ranges, distributions, and scenario comparisons.
10. Verify implementation with tests, invariants, and reproducible runs.
11. Review calibration and validation evidence for intended use.
12. Communicate limitations, governance conditions, and representation risk.
