# Cloud Computing and Algorithmic Infrastructure Governance Checklist

- Define computational purpose, workload, service owners, and critical dependencies.
- Version infrastructure as code and record active deployment artifacts.
- Review compute sizing, autoscaling thresholds, quotas, and capacity headroom.
- Document storage retention, backup, encryption, versioning, and data classification.
- Apply least privilege to users, service accounts, functions, pipelines, and deployment systems.
- Preserve logs, metrics, traces, alert history, and deployment records.
- Track cloud unit cost, managed-service cost, observability cost, and data-transfer cost.
- Test rollback, failover, backup restoration, degraded mode, and incident response.
- Map vendor dependencies and portability risks.
- Separate infrastructure availability from output validity, source quality, and governance approval.
