# Method Notes

This folder treats data structures and algorithm analysis as a joined history of representation and cost.

The core method is:

1. Identify the structure and the problem it solved.
2. List the operations the structure supports.
3. Compare memory layout and access patterns.
4. Analyze time and space cost.
5. Distinguish worst-case, average-case, and amortized behavior.
6. Trace how the structure appears in systems.
7. Examine safety and maintainability concerns.
8. Apply data-structure judgment to AI systems.

The goal is to avoid memorizing isolated structures and instead understand the historical relationship among representation, operation, scale, systems, and governance.
