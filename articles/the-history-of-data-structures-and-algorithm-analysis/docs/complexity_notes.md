# Complexity Notes

Algorithm analysis asks how cost grows as input size grows.

Important lenses:

- constant time: O(1);
- logarithmic time: O(log n);
- linear time: O(n);
- linearithmic time: O(n log n);
- quadratic time: O(n^2);
- exponential time: O(2^n);
- space complexity;
- worst-case analysis;
- average-case analysis;
- amortized analysis;
- recurrence relations;
- I/O and memory-locality analysis.

Big O is useful, but it is not everything. Constants, locality, cache behavior, allocation, storage, communication, and governance constraints also matter.
