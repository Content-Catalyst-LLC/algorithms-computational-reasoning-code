# Data Structure Notes

A data structure organizes information to support operations.

Core examples:

- arrays support indexed access and iteration;
- records group fields;
- linked lists support flexible insertion with traversal cost;
- stacks support nested control and LIFO access;
- queues support waiting and FIFO access;
- trees support hierarchy and ordered search;
- heaps support priority access;
- hash tables support expected fast key-value lookup;
- graphs support relation, dependency, and path reasoning;
- indexes support persistent query speed;
- abstract data types separate behavior from implementation.

Every structure creates tradeoffs among time, space, locality, flexibility, safety, and maintainability.
