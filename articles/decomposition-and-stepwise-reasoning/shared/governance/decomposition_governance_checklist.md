# Decomposition Governance Checklist

- Name the whole problem before splitting it.
- Identify subproblems and component responsibilities.
- Define boundaries, inputs, outputs, and assumptions.
- Map dependencies and data flows.
- Define sequence, branches, loops, and stopping conditions.
- Create unit, integration, edge-case, trace, and governance tests.
- Review whether parts recombine into a meaningful whole.
- Identify hidden dependencies, context loss, local optimization, and responsibility diffusion.
- Define monitoring, review, revision, and accountability processes.
