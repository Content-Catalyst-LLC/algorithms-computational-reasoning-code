# API and Interface Governance Checklist

- Identify each interface boundary and owner.
- Define responsibility, scope, inputs, outputs, errors, side effects, and permissions.
- Use schemas, types, and examples where practical.
- Version public or shared contracts.
- Test compatibility between providers and consumers.
- Track consumers before making breaking changes.
- Document rate limits, security scopes, data semantics, and operational limits.
- Preserve request IDs, logs, metrics, traces, and audit records.
- Review security, privacy, and least-privilege permissions.
- Maintain changelogs, deprecation timelines, and migration guides.
