# Compression, Encoding, and Information Efficiency Governance Checklist

- Define the information object: text, image, record, signal, stream, model output, archive, or token sequence.
- Define whether reconstruction must be exact.
- Choose and document encoding, schema, file format, and version.
- Choose and document compression method and loss policy.
- Preserve source, timestamp, schema, encoding, compression method, access, accessibility, and license metadata.
- Use checksums, hashes, validation rules, or error-detection mechanisms.
- Test interoperability across producing and consuming systems.
- Preserve accessibility metadata such as captions, alt text, language, and document structure where relevant.
- Evaluate storage, transmission, computational, and preservation trade-offs.
- Plan long-term lifecycle, migration, decoding, and format governance.
