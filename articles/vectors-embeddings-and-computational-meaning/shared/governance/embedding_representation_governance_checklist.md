# Embedding Representation Governance Checklist

- Define the embedded object: word, sentence, document, image, user, product, graph node, case, concept, or system state.
- Define the task: search, classification, clustering, recommendation, retrieval, comparison, or analysis.
- Document the embedding model, version, dimensionality, training context, and compatibility constraints.
- Preserve source ID, timestamp, preprocessing, chunking, access rules, and evaluation metadata.
- Define similarity metric and score interpretation.
- Validate nearest-neighbor and retrieval outcomes against actual use cases.
- Audit bias, coverage, language behavior, domain behavior, and stakeholder impacts.
- Monitor drift in data, model versions, vocabulary, sources, and retrieval outcomes.
- Enforce access boundaries before retrieval and display.
- Limit interpretation: vector similarity is a computational signal, not automatic truth or meaning.
