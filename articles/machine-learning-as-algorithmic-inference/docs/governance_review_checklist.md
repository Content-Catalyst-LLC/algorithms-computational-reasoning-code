# Governance Review Checklist

- Is the intended use clearly defined?
- Are the features valid for the purpose?
- Is the label a defensible measurement of the target construct?
- Has train/test leakage been ruled out?
- Does test performance hold under relevant deployment conditions?
- Are thresholds justified and documented?
- Are false positives and false negatives reviewed as different harms?
- Are subgroup errors reported?
- Are model scores calibrated enough for their intended interpretation?
- Is there a contestability, appeal, or human-review pathway?
- Are use boundaries explicit?
- Is monitoring planned for drift, decay, and feedback loops?
