# Control Flow Governance Checklist

- Map sequence, branch, loop, recursive, asynchronous, and error paths.
- Define branch criteria, thresholds, and review triggers.
- Define loop invariants, progress measures, and stopping conditions.
- Define recursive base cases, recursive cases, and depth limits.
- Document state updates, accumulators, queues, stacks, and visited sets.
- Test empty, extreme, cyclic, invalid, concurrent, delayed, and duplicate cases.
- Bound retries, timeouts, queue growth, recursion depth, and simulation horizon.
- Preserve traces through logs, correlation IDs, run manifests, and audit records.
- Review control-flow changes through version control and design records.
