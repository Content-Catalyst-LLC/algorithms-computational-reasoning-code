# Ada Lovelace, Programming, and the Imagination of Computation

This companion folder supports the Sustainable Catalyst article **Ada Lovelace, Programming, and the Imagination of Computation**.

It provides reproducible, teaching-oriented workflows for studying Ada Lovelace as a bridge between mechanical procedure and software imagination: the Analytical Engine, Charles Babbage, Luigi Menabrea, translation, Lovelace's Notes, Note G, Bernoulli numbers, punched cards, symbolic operations, computation beyond arithmetic, poetical science, music and pattern, machine limits, collaboration, authorship, gender and recognition, and AI-era caution.

## What this folder demonstrates

- how Lovelace interpreted the Analytical Engine as more than a calculator;
- how Note G can be studied as machine-oriented procedure without projecting modern software backward;
- how symbolic operations allow computation to extend beyond arithmetic;
- how the music analogy shows computational generality through representation and pattern;
- how the Lovelace objection clarifies machine limits and AI-era claims;
- how translation, notes, collaboration, and commentary can be intellectual authorship;
- how gender, credit, and recognition shape the memory of computing history;
- how to distinguish useful historical shorthand from slogan, myth, and dismissal.

## Repository structure

- `python/` — reference Lovelace computation-imagination map workflow and tests
- `r/` — programming and computational-imagination diagnostics
- `sql` — schema and queries for themes, imagination scores, and cautions
- `calculators/` — self-contained Bernoulli, operation-sequence, symbolic-generality, machine-limit, authorship, and myth-caution calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, Analytical Engine notes, Note G notes, symbolic operation notes, AI caution notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and interpretive. They are educational scaffolds for historical and conceptual analysis, not claims about a single final interpretation of Lovelace, Babbage, or programming history.
