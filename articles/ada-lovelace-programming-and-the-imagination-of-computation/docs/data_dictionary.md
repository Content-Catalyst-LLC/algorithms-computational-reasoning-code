# Data Dictionary

## lovelace_themes.csv

- `theme_id`: interpretive theme in the Lovelace computation-imagination map.
- `programming_structure`: importance of ordered procedure, operations, intermediate values, and machine orientation.
- `symbolic_generality`: degree to which the theme extends computation beyond arithmetic.
- `machine_orientation`: degree to which the theme relates procedure to an executing machine.
- `mathematical_grounding`: degree to which mathematics, Bernoulli numbers, operations, or formal relations matter.
- `imaginative_reach`: degree to which the theme imagines new computational possibility.
- `limit_awareness`: degree to which machine limits and dependency are visible.
- `collaboration`: degree to which Babbage, Menabrea, translation, notes, and networks matter.
- `authorship`: degree to which interpretation, annotation, and intellectual contribution are central.
- `modern_resonance`: degree to which the theme clarifies modern computing history.
- `ai_caution`: degree to which the theme clarifies modern AI agency, creativity, and overclaiming.

## lovelace_computation_imagination_map.csv

- `imagination_score`: average of the theme dimensions.
- `interpretive_status`: core, major, or supporting Lovelace computation thread.

## interpretation_cautions.csv

- `caution`: interpretive risk to avoid.
- `meaning`: explanation of the caution.
