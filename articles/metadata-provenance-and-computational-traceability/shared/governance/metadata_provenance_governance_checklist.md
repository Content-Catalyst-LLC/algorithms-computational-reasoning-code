# Metadata, Provenance, and Traceability Governance Checklist

- Define the object being traced: dataset, file, record, model, output, article, chart, decision, or workflow.
- Define required descriptive, structural, administrative, technical, provenance, quality, rights, and governance metadata.
- Use stable identifiers for objects, versions, activities, agents, and outputs.
- Record source, collection method, authority, and origin.
- Capture lineage from raw input through transformation and publication.
- Version data, code, schema, model, environment, parameters, and outputs.
- Add timestamps for creation, modification, processing, review, access, and publication.
- Use integrity markers such as checksums, signatures, validation logs, or append-only records.
- Apply access-control, privacy, retention, consent, and disclosure policies.
- Assign stewardship and a review cadence for metadata quality.
