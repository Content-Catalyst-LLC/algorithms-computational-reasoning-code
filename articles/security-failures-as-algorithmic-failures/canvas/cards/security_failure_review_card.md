# Security Failure Review Card

- What protected asset was affected?
- What procedure did the system actually follow?
- Which assumption failed under pressure?
- Where did data, identity, authority, state, or trust cross a boundary?
- Did authorization bind subject, resource, action, context, and time?
- Were cryptographic procedures and artifact integrity verified?
- Were dependencies, configuration, and secrets governed?
- Were logs and monitoring adequate for detection and reconstruction?
- What lifecycle or governance failure allowed the weakness to persist?
