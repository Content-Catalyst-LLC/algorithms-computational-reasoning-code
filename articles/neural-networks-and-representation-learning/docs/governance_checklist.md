# Governance Checklist for Neural Representations

- [ ] Define the model purpose and prohibited uses.
- [ ] Document input data provenance and measurement limits.
- [ ] Explain labels, objectives, and loss functions.
- [ ] Compare train, validation, test, and shifted-test behavior.
- [ ] Report calibration and uncertainty where relevant.
- [ ] Review subgroup performance and proxy encoding risks.
- [ ] Inspect embedding or representation behavior with domain experts.
- [ ] Document interpretability artifacts and their limits.
- [ ] Establish monitoring for drift, incidents, and model updates.
- [ ] Provide meaningful escalation and appeal routes when outputs affect people.
