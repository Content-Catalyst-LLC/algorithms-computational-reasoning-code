# Search Architecture Governance Checklist

- Define retrieval goals and user tasks.
- Inventory searchable content and exclusions.
- Decide retrieval units: document, section, passage, record, entity, or vector.
- Audit metadata completeness for titles, tags, dates, source fields, alt text, captions, and repository links.
- Document tokenization, normalization, synonym expansion, and query interpretation.
- Inventory indexed fields, facets, freshness signals, vector indexes, and ranking signals.
- Evaluate search quality with precision, recall, ranked metrics, zero-result queries, and expert review.
- Govern search logs, click feedback, retention, privacy, access, and reuse.
- Preserve provenance and citation links for retrieved results.
- Communicate search limitations, freshness, personalization, and correction paths.
