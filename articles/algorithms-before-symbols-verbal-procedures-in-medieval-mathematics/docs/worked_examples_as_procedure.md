# Worked Examples as Procedure

Worked examples are procedural infrastructure.

A worked example can provide:

1. A problem statement.
2. Identification of knowns and unknowns.
3. Ordered operations.
4. Intermediate states.
5. A final result.
6. A check or demonstration.
7. A reusable pattern for similar cases.

In modern terms, a worked example can act like a sample run, test case, documentation block, and teaching template.
