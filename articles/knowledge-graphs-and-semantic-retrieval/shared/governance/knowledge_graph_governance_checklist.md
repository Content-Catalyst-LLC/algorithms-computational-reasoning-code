# Knowledge Graph and Semantic Retrieval Governance Checklist

- Define graph purpose and retrieval tasks.
- Inventory entity types, edge types, attributes, and provenance fields.
- Define relationship vocabulary and ontology constraints.
- Review entity resolution for duplicates, aliases, acronyms, and ambiguous names.
- Separate asserted, inferred, predicted, deprecated, and disputed relationships.
- Preserve source evidence for important edges and paths.
- Test graph traversal, neighborhood retrieval, and path explanations.
- Evaluate semantic retrieval with entity, relationship, path, source, and task judgments.
- Audit dense-node bias, missing relationships, ontology drift, and stale graph state.
- Provide correction workflows for identity, relationship, provenance, and schema errors.
