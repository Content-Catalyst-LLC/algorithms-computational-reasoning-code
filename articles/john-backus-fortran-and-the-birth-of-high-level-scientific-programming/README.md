# John Backus, Fortran, and the Birth of High-Level Scientific Programming

This companion folder supports the Sustainable Catalyst article **John Backus, Fortran, and the Birth of High-Level Scientific Programming**.

It provides reproducible, teaching-oriented workflows for studying John Backus and FORTRAN as the practical birth of high-level scientific programming: formula translation, the IBM 704, compiler optimization, numerical computing, arrays, loops, subroutines, machine independence, portability, Backus-Naur Form, language specification, scientific software, high-performance computing, AI code-generation review, and responsible scientific software reasoning.

## What this folder demonstrates

- how formula translation let scientific users express mathematical procedures above machine code;
- why FORTRAN adoption depended on compiler optimization and performance credibility;
- how arrays, loops, and subroutines made numerical scientific work programmable at scale;
- how source-level portability changed scientific software continuity;
- why FORTRAN became scientific and high-performance computing infrastructure;
- how Backus-Naur Form connects Backus to formal language specification;
- why Backus's later work questioned the von Neumann style of programming;
- how Fortran's lesson applies to AI-generated scientific code: abstraction must earn trust through correctness, performance, tests, documentation, and review.

## Repository structure

- `python/` — reference Backus/FORTRAN scientific-programming map workflow and tests
- `r/` — Fortran legacy diagnostics and figures
- `sql/` — schema and queries for Backus themes, scores, examples, and cautions
- `calculators/` — self-contained formula-translation, vector-loop, portability, performance, translation-trust, BNF, numerical-stability, and AI-code caution calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, formula-translation notes, compiler optimization notes, Fortran/HPC notes, BNF notes, AI review notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not replacements for compiler theory, numerical analysis, benchmarking, safety review, or production scientific software validation.
