# Algorithm Design Governance Checklist

- Define the problem before selecting a procedure.
- Record valid inputs, outputs, constraints, assumptions, and success criteria.
- Choose representations and data structures with explicit rationale.
- Provide correctness reasoning or a proof sketch where appropriate.
- Define progress measure and stopping condition.
- Estimate time, space, communication, and implementation costs.
- Test ordinary cases, boundary cases, invalid cases, and adversarial cases.
- Document robustness limits and failure behavior.
- Preserve design evidence, run outputs, test results, and review notes.
- Review context, consequences, interpretation, and human oversight needs.
