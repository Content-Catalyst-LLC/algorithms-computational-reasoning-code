# Documentation, Reproducibility, and Debugging Notes

Al-Jazarī’s importance depends partly on technical documentation.

A diagram can show:

- component placement;
- hidden mechanisms;
- flow paths;
- motion transfer;
- event sequence;
- reset structure.

Mechanical debugging asks where the intended sequence fails:

- input;
- state;
- trigger;
- transition;
- output;
- reset;
- material constraint.

Documentation makes action reproducible and inspectable.
