# Al-Jazarī, Automata, and Sequenced Mechanical Action

This companion folder supports the Sustainable Catalyst article **Al-Jazarī, Automata, and Sequenced Mechanical Action**.

It provides reproducible, teaching-oriented workflows for mapping al-Jazarī’s automata as sequenced mechanical action: clocks, water timing, elephant clock logic, cams, gears, levers, floats, vessels, water-raising machines, musical automata, conditional triggers, repeatable cycles, documentation, implementation limits, and embodied algorithmic reasoning.

## What this folder demonstrates

- how clocks transform gradual material change into visible timed events;
- how automata can embody ordered action without modern software or electronics;
- how cams, levers, gears, floats, vessels, and weights encode mechanical control;
- how repeated cycles resemble loop-like behavior while remaining materially different from software loops;
- how diagrams, construction notes, and operational descriptions support reproducibility;
- how implementation constraints such as leakage, friction, calibration, wear, and reset conditions matter;
- how al-Jazarī should be framed as a major engineer of sequenced mechanical action, not as an inventor of modern robots, digital computers, or AI.

## Repository structure

- `python/` — reference sequenced mechanical action map workflow and tests
- `r/` — automata and sequenced-action diagnostics and figures
- `sql` — schema and queries for sequenced-action themes, scores, and interpretation cautions
- `calculators/` — self-contained event-trigger, cycle-time, cam-schedule, repeat-cycle, water-flow, reset-state, and sequenced-action score calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, clock/automata notes, cams/levers notes, documentation/reproducibility notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and interpretive. They are educational scaffolds for historical and conceptual analysis, not engineering, safety, construction, or device-operation instructions.
