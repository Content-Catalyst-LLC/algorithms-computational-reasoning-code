# Runtime Context Governance Checklist

- Identify the runtime, compiler, interpreter, VM, database engine, browser, or managed platform.
- Record operating system, hardware architecture, resource limits, and deployment platform.
- Pin dependencies, toolchains, models, datasets, and container images.
- Validate configuration, environment variables, secrets, feature flags, and paths.
- Map external APIs, databases, queues, model endpoints, object stores, and file systems.
- Define observability: logs, metrics, traces, profiles, audit records, and error reports.
- Review permissions using least privilege.
- Preserve runtime manifests, checksums, output records, and run logs.
- Test portability across supported environments.
- Treat runtime context as part of the computational artifact.
