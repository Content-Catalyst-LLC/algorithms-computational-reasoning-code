# Method Notes

This folder treats Church as a bridge between functions, formal calculation, lambda reduction, undecidability, type theory, and programming-language foundations.

The core method is:

1. Separate Church's lambda calculus from Turing's machine model.
2. Define the syntax: variables, abstraction, and application.
3. Trace beta reduction as computation by substitution.
4. Analyze alpha conversion and variable capture.
5. Study Church numerals as encoding.
6. Connect recursion and fixed points to nontermination.
7. Read Church's thesis as a bridge between informal calculability and formal models.
8. Connect lambda calculus to type theory, functional programming, proof assistants, and formal reasoning.

The goal is to understand computation as symbolic function transformation, not only as machine motion.
