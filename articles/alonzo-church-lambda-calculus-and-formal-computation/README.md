# Alonzo Church, Lambda Calculus, and Formal Computation

This companion folder supports the Sustainable Catalyst article **Alonzo Church, Lambda Calculus, and Formal Computation**.

It provides reproducible, teaching-oriented workflows for studying Church's contribution to formal computation: lambda calculus, functions, abstraction, application, variables, binding, scope, substitution, beta reduction, alpha conversion, eta conversion, normal forms, Church numerals, recursion, fixed points, effective calculability, Church's theorem, the Entscheidungsproblem, Church-Turing thesis, type theory, functional programming, programming-language semantics, proof assistants, and AI-era caution.

## What this folder demonstrates

- how Church models computation without machines;
- how lambda calculus represents computation as functions, abstraction, application, substitution, and reduction;
- how variable binding, scope, alpha conversion, and capture avoidance matter for formal reasoning;
- how Church numerals show data encoding inside a formal system;
- how recursion and fixed points reveal expressive power and nontermination;
- how Church's theorem and the Entscheidungsproblem mark formal limits of decision procedure;
- how Church and Turing converge on effective computability through different images of computation;
- how lambda calculus informs type theory, functional programming, proof systems, and AI-era caution.

## Repository structure

- `python/` — reference Church formal-computation map workflow and tests
- `r/` — formal-computation diagnostics and figures
- `sql/` — schema and queries for Church themes, scores, and cautions
- `calculators/` — self-contained beta-reduction, alpha-conversion, Church numeral, normal-form, fixed-point, decidability, and AI-caution calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, lambda-calculus notes, binding/scope notes, reduction notes, computability notes, type-theory notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and interpretive. They are educational scaffolds for historical and conceptual analysis, not replacements for formal lambda-calculus proofs, type theory, or specialist computability scholarship.
