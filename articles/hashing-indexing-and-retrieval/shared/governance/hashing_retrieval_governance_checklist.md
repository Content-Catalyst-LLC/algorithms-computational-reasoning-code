# Hashing, Indexing, and Retrieval Governance Checklist

- Define what the system must retrieve: records, documents, cases, files, objects, claims, entities, or evidence.
- Define keys, terms, fields, fingerprints, or queries used for retrieval.
- Document hash function purpose: ordinary lookup, fingerprinting, integrity, partitioning, or security-sensitive use.
- Define collision, ambiguity, and equality handling.
- Document index coverage, exclusions, stale entries, and pending items.
- Preserve source, timestamp, version, transformation history, confidence, and access metadata.
- Define cache expiration, invalidation, eviction, and freshness checks.
- Explain ranking signals and limitations.
- Apply access controls to both index and retrieval result layers.
- Govern changes in keys, indexes, caches, ranking models, metadata, and downstream reuse.
