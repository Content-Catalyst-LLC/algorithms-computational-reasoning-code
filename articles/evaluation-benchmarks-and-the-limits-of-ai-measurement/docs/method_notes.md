# Method Notes

This folder treats AI evaluation as an accountable measurement system.

The core method is:

1. Define intended use and stakes.
2. Select benchmarks and explain why they fit the use case.
3. Document test data, scoring rules, and metric limits.
4. Run capability, calibration, safety, and robustness checks.
5. Disaggregate performance across relevant tasks, groups, languages, and contexts.
6. Identify benchmark saturation, contamination, and gaming risks.
7. Connect evaluation to deployment monitoring, approval, escalation, and use boundaries.

The examples are dependency-light and educational.
