# Explanation Quality Notes

An explanation should be evaluated, not merely displayed.

Important quality questions:

- Is it faithful to the actual system behavior?
- Is it stable across similar cases?
- Can the intended audience understand it?
- Does it support action, correction, review, or recourse?
- Does it include uncertainty and limits?
- Does it avoid false precision?
- Does it connect to data provenance and institutional process?
- Does it support contestability and remedy?

Explanations can be misleading when they are plausible, generic, unfaithful, unactionable, or disconnected from governance.
