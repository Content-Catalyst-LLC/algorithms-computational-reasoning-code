# Method Notes

This folder treats explanation as a governance artifact.

The core method is:

1. Define the audience and action purpose.
2. Identify what must be transparent: data, model, evaluation, workflow, versions, limits, and governance.
3. Choose explanation methods: local, global, feature attribution, counterfactual, example-based, uncertainty, or process explanation.
4. Test explanation quality: faithfulness, stability, understandability, actionability, completeness, and uncertainty communication.
5. Connect explanations to contestability, correction, appeal, and remedy.
6. Monitor explanation quality and explanation drift after deployment.
7. Update documentation, governance, and remediation when explanations reveal failure modes.

The examples are dependency-light and educational.
