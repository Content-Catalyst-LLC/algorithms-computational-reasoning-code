# Transparency, Explainability, and Interpretability

This companion folder supports the Sustainable Catalyst article **Transparency, Explainability, and Interpretability**.

It provides reproducible, teaching-oriented workflows for transparency, explainability, interpretability, documentation, model cards, datasheets, local explanations, global explanations, feature attribution, counterfactual explanation, uncertainty communication, contestability, auditability, governance, remediation, and responsible algorithmic interpretation.

## Article sequence

- Previous: `algorithmic-bias-data-and-institutional-history`

## What this folder demonstrates

- how transparency, explainability, and interpretability differ;
- how explanation quality can be reviewed through faithfulness, stability, understandability, actionability, and uncertainty communication;
- how transparency capacity depends on documentation, governance readiness, and uncertainty communication;
- how contestability and remediation determine whether explanations support accountability;
- how explanation artifacts can be turned into audit outputs, calculators, governance registers, and Canvas-ready cards.

## Repository structure

- `python/` — reference explanation and transparency audit workflow and tests
- `r/` — diagnostic summaries and figures
- `sql` — schema and governance queries for explanation artifacts, transparency capacity, and contestability
- `calculators/` — self-contained explanation-quality, transparency-capacity, accountability-capacity, and explanation-risk calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, explanation-quality notes, governance review, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not legal, compliance, audit certification, civil-rights, medical, credit, employment, public-benefits, or operational advice.
