# Cryptographic Review Checklist

- Define security goals: confidentiality, integrity, authentication, freshness, nonrepudiation, and availability.
- State the threat model and adversary capabilities.
- Use approved protocols and reviewed libraries.
- Inventory keys, secrets, certificates, and trust anchors.
- Verify certificate-validation and signature-verification paths.
- Avoid custom cryptography and undocumented protocol changes.
- Review nonce, IV, randomness, and token-generation rules.
- Document rotation, revocation, incident response, and deprecation paths.
- Communicate what is protected, what is not protected, and under which assumptions.
