# Governance Checklist for LLM Procedural Reasoning

- [ ] Define intended use and prohibited use.
- [ ] Classify task stakes before deployment.
- [ ] Specify source requirements for factual claims.
- [ ] Record prompt, context, and retrieved evidence.
- [ ] Track tool calls, permissions, and action boundaries.
- [ ] Verify calculations, code, citations, and factual claims.
- [ ] Require expert review for high-stakes outputs.
- [ ] Preserve appeal or contestability when outputs affect people.
- [ ] Monitor hallucinations, prompt injection, and misuse patterns.
- [ ] Maintain incident response and update procedures.
