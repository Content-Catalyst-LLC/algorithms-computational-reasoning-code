# Prompt Review Notes

Prompts should be treated as procedural design artifacts. A strong prompt should include:

- task definition;
- audience and use case;
- permitted sources;
- output format;
- constraints and boundaries;
- uncertainty expectations;
- citation requirements;
- escalation conditions;
- tool-use permissions.

Prompt review should also check for ambiguity, missing context, hidden assumptions, unsafe delegation, and conflicts between user goals and institutional policy.
