# Method Notes

This folder treats failure modes as design and governance objects.

1. Define the system boundary across data, model, interface, workflow, infrastructure, institution, and affected environment.
2. Identify plausible failure modes.
3. Score likelihood, severity, detectability, and controllability.
4. Prioritize failures by risk and detectability.
5. Evaluate resilience: monitoring, fallback, rollback, escalation, and repair.
6. Assign owners and escalation pathways.
7. Monitor incidents, appeals, overrides, drift, and recurrence.
8. Update controls after incidents and near misses.
