# When Algorithms Should Not Be Used

This companion folder supports the Sustainable Catalyst article **When Algorithms Should Not Be Used**.

It provides reproducible, teaching-oriented workflows for algorithmic non-use, refusal, restraint, inappropriate targets, illegitimate proxies, high-stakes harm, irreversibility, contestability, governance capacity, human judgment, repairability, alternatives analysis, pause decisions, rollback decisions, retirement decisions, and responsible algorithmic boundaries.

## Article sequence

- Previous: `documentation-model-cards-and-datasheets-for-algorithms`
- Current: `when-algorithms-should-not-be-used`
- Series status: final article in the current Algorithms & Computational Reasoning sequence

## What this folder demonstrates

- how non-use decisions can be documented as responsible governance;
- how non-use pressure can combine stakes, irreversibility, governance weakness, and proxy illegitimacy;
- how responsible-use readiness can combine target legitimacy, data legitimacy, contestability, human judgment, governance capacity, and repairability;
- how refusal, restraint, support-only, delay, pause, rollback, and retirement decisions can be represented as audit artifacts;
- how governance artifacts, calculators, tests, SQL queries, Canvas cards, and multilingual examples can support algorithmic restraint.

## Repository structure

- `python/` — reference algorithmic non-use review workflow and tests
- `r/` — diagnostic summaries and figures
- `sql/` — schema and queries for non-use review, refusal criteria, responsible-use readiness, and governance actions
- `calculators/` — self-contained non-use pressure, responsible-use readiness, repairability, and refusal-threshold calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, non-use register notes, alternatives review, governance review, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not legal, compliance, audit certification, civil-rights, medical, credit, employment, public-benefits, or operational advice.
