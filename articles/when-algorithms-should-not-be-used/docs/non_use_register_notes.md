# Non-Use Register Notes

A non-use register records decisions not to build, buy, deploy, scale, continue, or rely on algorithmic systems.

Useful fields include:

- decision identifier;
- proposed system or use case;
- decision type: reject, delay, limit, support-only, pause, rollback, retire;
- affected people and stakes;
- target legitimacy concerns;
- data legitimacy concerns;
- proxy concerns;
- contestability concerns;
- governance capacity concerns;
- evidence reviewed;
- non-algorithmic alternatives;
- decision owner;
- conditions for reconsideration.

A non-use register protects institutional memory and prevents inappropriate systems from reappearing under new labels.
