# Formalization Governance Checklist

- Separate the real-world concern from the formal computational task.
- Define inputs, outputs, constraints, states, objectives, and assumptions.
- Document what the representation includes and excludes.
- Test edge cases, invalid inputs, ties, missing data, and no-solution cases.
- Define stopping conditions and failure reporting.
- Evaluate whether the objective matches the intended purpose.
- Explain how outputs should and should not be interpreted.
- Define review, appeal, monitoring, revision, and retirement criteria.
