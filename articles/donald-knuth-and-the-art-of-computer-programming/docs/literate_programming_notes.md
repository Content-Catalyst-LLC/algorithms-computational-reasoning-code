# Literate Programming Notes

Literate programming treats a program as a document of reasoning for human readers and a source of executable code for machines.

Key ideas:

- explanation is not an afterthought;
- code can be organized around human understanding;
- one source can produce both documentation and compilable code;
- identifiers, modules, cross-references, and indexes support maintenance;
- the program should be readable as a technical essay.

This is a standard for responsible software explanation.
