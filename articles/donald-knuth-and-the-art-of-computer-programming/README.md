# Donald Knuth and the Art of Computer Programming

This companion folder supports the Sustainable Catalyst article **Donald Knuth and the Art of Computer Programming**.

It provides reproducible, teaching-oriented workflows for studying Donald Knuth's algorithmic legacy: The Art of Computer Programming, algorithm analysis, asymptotic notation, sorting, searching, data structures, random numbers, seminumerical algorithms, MIX, MMIX, TeX, METAFONT, digital typography, literate programming, WEB, CWEB, mathematical exposition, exercises, errata culture, software readability, AI code-generation review, and responsible algorithmic explanation.

## What this folder demonstrates

- how algorithmic analysis compares procedures by cost, assumptions, and growth;
- how asymptotic notation helps distinguish small-example speed from structural scalability;
- how sorting, searching, data structures, and random-number generation reveal deep algorithmic reasoning;
- how MIX/MMIX connect abstract algorithms to executable machine models;
- how TeX and METAFONT connect computation, typography, and technical clarity;
- how literate programming makes source code a document of reasoning;
- why generated code is not automatically understood, analyzed, tested, or maintainable;
- how Knuth's standard links proof, exposition, historical context, and craft.

## Repository structure

- `python/` — reference Knuth algorithmic-art map workflow and tests
- `r/` — Knuthian analysis diagnostics and figures
- `sql/` — schema and queries for Knuth themes, scores, and cautions
- `calculators/` — self-contained art-score, asymptotic growth, sorting lower-bound, runtime comparison, literate-programming split, TeX clarity, documentation, and AI code-review caution calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, algorithm-analysis notes, literate-programming notes, TeX/METAFONT notes, AI review notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not replacements for formal algorithm analysis, benchmarking, security review, or production software certification.
