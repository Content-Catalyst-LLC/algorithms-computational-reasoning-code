# Grace Hopper, Compilers, and the Humanization of Programming

This companion folder supports the Sustainable Catalyst article **Grace Hopper, Compilers, and the Humanization of Programming**.

It provides reproducible, teaching-oriented workflows for studying Grace Hopper's programming-language legacy: compilers, automatic programming, A-0, A-2, UNIVAC, FLOW-MATIC, COBOL, machine-independent languages, human-readable source code, compiler diagnostics, debugging culture, documentation, programming-language standards, compiler validation, business data processing, institutional computing, AI code-generation caution, and responsible software automation.

## What this folder demonstrates

- how compilers mediate between human-facing source language and machine-executable code;
- how automatic programming reframed computers as tools that could help people program computers;
- how subroutine libraries and symbolic calls made programming compositional;
- how FLOW-MATIC and COBOL made business computation more readable and institutionally usable;
- how machine independence and portability changed software development;
- how documentation, debugging, standards, and validation made programming maintainable and governable;
- why human-readable programming still requires rigor, tests, specifications, and review;
- how Hopper's legacy clarifies the difference between formal compilation and AI code generation.

All datasets and examples are synthetic and educational. They are not replacements for compiler theory, programming-language semantics, software engineering, standards conformance, security review, or AI coding-tool governance.
