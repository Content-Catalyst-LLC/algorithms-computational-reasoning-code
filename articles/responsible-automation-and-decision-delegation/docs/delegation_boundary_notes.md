# Delegation Boundary Notes

Responsible automation depends on explicit boundaries.

Delegation boundary questions:

- What exact task is being automated?
- Is the system organizing information, recommending action, routing cases, or making final decisions?
- Which cases require human review?
- Which uncertainty signals should pause automation?
- Which high-stakes decisions require escalation?
- Which outcomes are reversible?
- Which affected people need notice, reasons, and appeal?
- Who can pause, rollback, modify, or retire the system?

Automation should expand only after evidence, contestability, and governance capacity improve.
