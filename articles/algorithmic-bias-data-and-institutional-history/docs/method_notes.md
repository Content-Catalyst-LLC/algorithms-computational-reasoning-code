# Method Notes

This folder treats algorithmic bias as a historical, technical, and institutional issue.

The core method is:

1. Map data provenance and institutional origin.
2. Identify affected groups, contexts, and missing populations.
3. Review representation gaps between data and deployment.
4. Review labels, proxies, and measurement validity.
5. Compute selection, representation, label, and historical-risk indicators.
6. Examine feedback loops and deployment context.
7. Evaluate contestability, correction, remediation, and governance ownership.
8. Update systems, policies, datasets, and monitoring after findings.

The examples are dependency-light and educational.
