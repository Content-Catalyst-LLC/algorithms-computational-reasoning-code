# Graph Relationship Governance Checklist

- Define the node type: entities, states, tasks, documents, claims, places, concepts, or components.
- Define the exact edge meaning: connection, dependency, citation, similarity, influence, flow, transition, or causality.
- Decide whether edges are directed, undirected, typed, weighted, temporal, or uncertain.
- Document units and meaning for edge weights.
- Preserve provenance, source, timestamp, confidence, version, and validity period.
- Select the graph representation: adjacency list, adjacency matrix, edge list, property graph, or knowledge graph.
- Select algorithms only after edge semantics are clear.
- Audit path interpretation, especially for long paths.
- Review relationship-overclaim risk: causality, authority, centrality, influence, or certainty.
- Govern changes in nodes, edges, weights, metadata, and downstream reuse.
