# The Unknown, the Variable, and Islamic Mathematical Philosophy

This companion folder supports the Sustainable Catalyst article **The Unknown, the Variable, and Islamic Mathematical Philosophy**.

It provides reproducible, teaching-oriented workflows for mapping the unknown as a disciplined mathematical object: thing, root, square, number, case classification, al-jabr, al-muqābalah, geometric demonstration, verbal algebra, practical problem solving, translation into Latin and vernacular traditions, the gradual emergence of variable thinking, and the philosophical status of procedure.

## What this folder demonstrates

- how an unknown becomes a named mathematical object;
- how verbal algebra can be rigorous without modern symbolic notation;
- how thing, root, square, and number organize algebraic reasoning;
- how equation cases function as procedural dispatch rules;
- how restoration, balancing, reduction, completion, extraction, and verification transform problems;
- how geometric demonstration relates algebraic procedure to proof;
- how terms such as Arabic *shayʾ*, Latin *res*, and Italian *cosa* help transmit the unknown across traditions;
- how the unknown differs from the modern variable while helping prepare variable thinking;
- how practical problems can raise philosophical questions about number, magnitude, abstraction, proof, and method.

## Repository structure

- `python/` — reference unknown-variable concept map workflow and tests
- `r/` — unknown, variable, and philosophy diagnostics and figures
- `sql/` — schema and queries for unknown-variable themes, scores, and interpretation cautions
- `calculators/` — self-contained unknown-score, equation-case, square-completion, verification, translation-continuity, and variable-transition calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, unknown vocabulary notes, geometric demonstration notes, translation notes, philosophy notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and interpretive. They are educational scaffolds for historical and conceptual analysis, not claims about a single manuscript lineage or universal algebraic chronology.
