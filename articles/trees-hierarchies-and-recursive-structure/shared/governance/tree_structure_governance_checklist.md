# Tree Structure Governance Checklist

- Define the root and the domain boundary.
- Define what edges mean: containment, dependency, classification, derivation, routing, or authority.
- Test whether each child truly belongs under one parent.
- Identify leaves, terminal outcomes, and unresolved or ambiguous cases.
- State invariants: no cycles, valid parent links, ordering, heap property, prefix paths, or decision routing.
- Select traversal methods and document their interpretation.
- Review depth, height, balance, and branch visibility.
- Track cross-links and non-tree relationships.
- Audit whether hierarchy implies authority, priority, category boundaries, or naturalness.
- Govern changes in categories, relationships, split rules, path definitions, and downstream reuse.
