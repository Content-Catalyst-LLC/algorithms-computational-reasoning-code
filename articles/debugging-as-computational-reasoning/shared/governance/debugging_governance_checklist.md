# Debugging Governance Checklist

- State expected behavior before changing code.
- Reproduce the problem with the smallest useful case.
- Preserve inputs, environment, versions, logs, traces, and observed behavior.
- Form a testable hypothesis before applying a fix.
- Isolate the cause through reduced examples or targeted inspection.
- Verify that the fix addresses the cause, not merely the symptom.
- Add regression tests for every meaningful bug.
- Document the failure mode, assumption, fix, and future monitoring need.
- Monitor related systems for recurrence.
- Escalate consequential failures when human review, governance, or public communication is required.
