# Debugging as Computational Reasoning

Companion code and reproducible workflows for **Debugging as Computational Reasoning**.

This article examines debugging as a disciplined reasoning process: comparing expected and observed behavior, forming hypotheses, inspecting traces, isolating causes, verifying fixes, adding regression tests, and documenting lessons.

## Folder structure

```text
python/      debugging-reasoning audits, CLI tools, calculators, tests
r/           statistical summaries, visualization, reports
julia/       debugging trace simulations
sql/         schemas, failure traces, debugging queries
haskell/     typed debugging cases and failure reasoning
rust/        systems-style debugging diagnostics
go/          CLI/service-style debugging workflows
c/           low-level debugging examples
cpp/         high-performance debugging examples
fortran/     numerical failure diagnostics
java/        object-oriented debugging audit systems
typescript/ interface-facing debugging checks
prolog/      rule-based debugging logic
racket/      interpreter-style debugging examples
docs/        methodology, governance, and article notes
data/        synthetic datasets only
outputs/     generated outputs
notebooks/   notebook-ready walkthroughs
canvas/      Catalyst Canvas-ready cards and manifests
shared/      schemas, templates, taxonomies, benchmarks, governance
```

## Run

```bash
make smoke
make all
```

The workflows use synthetic data only and are intended for educational, reproducible article support.
