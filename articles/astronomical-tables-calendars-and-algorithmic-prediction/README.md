# Astronomical Tables, Calendars, and Algorithmic Prediction

This companion folder supports the Sustainable Catalyst article **Astronomical Tables, Calendars, and Algorithmic Prediction**.

It provides reproducible, teaching-oriented workflows for mapping astronomical tables, calendars, zīj traditions, ephemerides, interpolation, correction rules, cycles, timekeeping, manuscript transmission, predictive uncertainty, and algorithmic prediction before modern computers.

## What this folder demonstrates

- how astronomical tables function as stored computation;
- how calendars operate as rule-governed time systems;
- how lookup, interpolation, correction, and rounding support predictive procedure;
- how zīj traditions combine tables, model parameters, instructions, and transmission;
- how observation, model, table, procedure, and interpretation form a computational system;
- how prediction depends on assumptions, uncertainty, correction, and institutional use;
- how astronomical computation connects ritual time, administration, agriculture, navigation, and education.

## Repository structure

- `python/` — reference astronomical prediction table map workflow and tests
- `r/` — astronomical prediction diagnostics and figures
- `sql/` — schema and queries for prediction themes, scores, and interpretation cautions
- `calculators/` — self-contained table lookup, interpolation, cycle, calendar offset, correction, ephemeris-step, and prediction-score calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, zīj notes, interpolation/correction notes, calendar notes, uncertainty notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and interpretive. They are educational scaffolds, not astronomical, calendrical, religious, navigational, or legal advice.
