# Method Notes

This folder treats astronomical tables and calendars as algorithmic prediction systems.

The core method is:

1. Identify the table type: calendar, solar, lunar, planetary, trigonometric, geographic, or timekeeping.
2. Determine the input argument: date, degree, cycle, location, or observed quantity.
3. Identify columns, rows, units, parameters, and reference epoch.
4. Extract the rule of use: lookup, correction, interpolation, conversion, or combination.
5. Track how uncertainty is handled: rounding, correction, comparison, or revision.
6. Identify institutional purpose: ritual time, calendar, navigation, administration, teaching, or scholarship.
7. Ask how the table was copied, corrected, translated, or localized.
8. Compare with modern computation only after understanding the historical table-procedure system.

The goal is to treat tables as computational artifacts rather than passive lists.
