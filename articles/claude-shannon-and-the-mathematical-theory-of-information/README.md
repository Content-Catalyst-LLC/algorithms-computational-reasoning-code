# Claude Shannon and the Mathematical Theory of Information

This companion folder supports the Sustainable Catalyst article **Claude Shannon and the Mathematical Theory of Information**.

It provides reproducible, teaching-oriented workflows for studying Shannon's mathematical theory of information: entropy, uncertainty, probability, bits, binary coding, source coding, channel coding, channel capacity, noisy channels, redundancy, compression, error correction, signal, noise, communication systems, semantic boundaries, secrecy systems, digital circuits, Boolean algebra, data systems, AI infrastructure, and responsible information governance.

## What this folder demonstrates

- how Shannon made information measurable as uncertainty;
- how entropy links probability, surprise, redundancy, and compression;
- how source coding turns efficient representation into an algorithmic problem;
- how noisy channels and capacity define reliable communication boundaries;
- how error correction uses structured redundancy for reliability;
- how secrecy and cryptography can be analyzed as information-flow problems;
- how Shannon's work connects digital circuits, Boolean logic, communication, and computation;
- how AI systems depend on encoding, uncertainty, compression, transmission, and information governance;
- why Shannon information must not be confused with meaning, truth, wisdom, or responsible action.

## Repository structure

- `python/` — reference Shannon information theory map workflow and tests
- `r/` — information diagnostics and figures
- `sql/` — schema and queries for information-theory themes, scores, and cautions
- `calculators/` — self-contained entropy, information-content, mutual-information, channel-capacity, compression-bound, redundancy, error-correction, and semantic-boundary calculators
- `canvas/` — Canvas-ready cards, schemas, and exports
- `docs/` — method notes, data dictionary, entropy notes, channel notes, semantic boundary notes, AI governance notes, and use boundaries
- language folders — compact examples across Python, R, Julia, SQL, Haskell, Rust, Go, C, C++, Fortran, Java, TypeScript, Prolog, and Racket

All datasets and examples are synthetic and educational. They are not replacements for information theory, coding theory, statistics, communication engineering, cryptography, or governance review.
