# Redundancy, Compression, and Error Correction Notes

Redundancy has two major roles:

1. Remove redundancy for compression.
2. Add structured redundancy for reliability.

This is not a contradiction. Redundancy is a design variable.

Examples:

- source coding removes predictable structure;
- error-correcting codes add recoverable structure;
- cryptography may seek to hide redundancy;
- governance may audit redundancy, duplication, and bias.
