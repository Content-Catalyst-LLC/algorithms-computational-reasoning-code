# Reproducible Workflow Review Checklist

Use this checklist when reviewing a computational experiment or reproducible workflow.

1. Define the claim, output, or decision the workflow supports.
2. Identify source data and preserve raw inputs where possible.
3. Document every data transformation from raw to analysis-ready form.
4. Record parameters, scenario settings, thresholds, tolerances, and random seeds.
5. Preserve the exact code path that generated the outputs.
6. Record language versions, packages, compilers, operating system, and runtime context.
7. Generate output manifests linking files to inputs, parameters, code, and environment.
8. Include validation, testing, diagnostics, warnings, and sensitivity checks.
9. Preserve logs and avoid overwriting prior outputs without run identity.
10. State interpretation limits, governance boundaries, and review responsibility.
