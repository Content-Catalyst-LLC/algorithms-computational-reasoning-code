# Boundary Governance Checklist

- Define accepted inputs, rejected inputs, source, units, types, missingness, and validity rules.
- Define outputs, output meaning, uncertainty, confidence, and interpretation limits.
- Define states, valid state values, invalid states, and required history.
- Define allowed transitions, forbidden transitions, and transition validation.
- Define preconditions, postconditions, invariants, and side effects.
- Define stopping conditions, no-solution behavior, escalation, and handoff rules.
- Test ordinary cases, edge cases, invalid input, no-solution cases, and failure cases.
- Log versions, state transitions, errors, warnings, and review notes.
- Define monitoring, review, appeal, revision, and retirement processes.
