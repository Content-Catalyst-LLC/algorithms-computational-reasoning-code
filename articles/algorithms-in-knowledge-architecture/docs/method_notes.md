# Method Notes

This folder treats knowledge architecture as a living system rather than a static index.

The core method is:

1. Define the knowledge system's purpose, audience, scope, and governance needs.
2. Design taxonomy, metadata schema, relationship types, and editorial standards.
3. Build search, link, recommendation, and maintenance workflows.
4. Evaluate metadata completeness, search quality, link quality, recommendation quality, and provenance.
5. Review representation risk, category bias, ranking effects, and recommendation narrowing.
6. Define human editorial review, revision logs, deprecation rules, and provenance records.
7. Monitor gaps, stale content, broken links, user needs, and knowledge drift.

The examples are dependency-light and educational.
