# Governance Review

Algorithmic knowledge architecture governance should include:

- taxonomy register;
- metadata schema and validation rules;
- search evaluation records;
- internal-link audits;
- recommendation review;
- provenance and source records;
- freshness and maintenance workflows;
- duplicate and stale-content detection;
- human editorial approval;
- representation-risk review;
- deprecation and revision logs.

A knowledge system should be able to explain why something is categorized, surfaced, linked, recommended, updated, or retired.
