# Computability Governance Checklist

- State the exact procedural claim.
- Define the input domain and excluded cases.
- Distinguish computation, decision, recognition, approximation, and assistance.
- State whether the procedure guarantees termination.
- Identify whether no results, unknown results, and negative results are distinct.
- Check whether the problem resembles known undecidable problems.
- Document reductions, restrictions, boundedness, or approximations.
- State automation scope clearly.
- Preserve traces, assumptions, false positives, false negatives, and unresolved cases.
- Create escalation paths for undecidable, ambiguous, high-stakes, or out-of-domain cases.
