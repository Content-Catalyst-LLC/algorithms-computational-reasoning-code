# Software Architecture Governance Checklist

- Identify major modules, components, services, stores, jobs, and workflows.
- Define responsibilities and owners for each major component.
- Map dependencies, data flow, control flow, and runtime topology.
- Review boundaries, cohesion, coupling, and dependency direction.
- Identify where state lives and who owns authoritative truth.
- Define reliability patterns: retries, timeouts, idempotency, fallbacks, rollback, and dead-letter handling.
- Review security boundaries, permissions, secrets, and sensitive data access.
- Preserve observability through logs, metrics, traces, audit records, and deployment records.
- Record major design choices with architecture decision records.
- Review architecture drift, technical debt, deprecations, and migration paths.
